# 基于机器学习的校园图书借阅数据分析与阅读推荐

## 📖 项目简介
本项目基于真实校园图书馆借阅日志，构建了一套端到端的数据分析与智能推荐系统。项目涵盖数据清洗、探索性数据分析 (EDA)、引入时间衰减因子的特征工程、以及基于隐式反馈的混合协同过滤推荐模型构建，并最终实现了一个基于 Flask 的 Web 交互式可视化展示界面。

## 💻 技术栈
* **核心编程语言**：Python 3.9+
* **数据处理与科学计算**：Pandas, NumPy, SciPy
* **机器学习与核心算法**：Scikit-learn (TF-IDF, TruncatedSVD, 交叉验证与网格搜索)
* **探索性数据分析 (EDA)**：Matplotlib, Seaborn
* **Web 后端与服务接口**：Flask, Jinja2
* **前端交互与可视化**：HTML5, CSS3

## 📊 数据集说明
**东北财经大学图书馆用户借阅记录数据集**
* **数据来源**：https://www.libraryjournal.com.cn/CN/Y2022/V41/I10/104
* **数据规模**：包含 10,676 名真实用户和 52,742 本有效馆藏图书。
* **数据特征**：涵盖用户维度（USERID, SEX, DEPT, OCCUPATION 等）与图书维度（BOOK_ID, CALL_NO, TITLE, ABSTRACT, SUB 等），数据维度丰富，利于深度信息挖掘。

## 🛠️ 数据预处理 (Data Cleaning)
真实系统导出的数据往往存在大量空缺与噪音，本项目执行了严格的清洗策略：
1. **缺失值处理**：将性别、学院等用户画像属性的缺失值统一填充为 `'Unknown'`；摘要、主题词等文本特征缺失值填充为空字符串，避免影响 TF-IDF 向量化。
2. **时间格式修复**：利用正则表达式修复原始数据中混杂的错误时间格式（如 `2020-01-1321:23:17`），并统一转化为 `datetime` 对象。
3. **逻辑异常值过滤**：剔除借阅时间为空的无效记录；剔除借阅时长 $\le 0$ 的“秒借秒还”误操作记录；通过特征去重剔除系统卡顿导致的重复打点数据。
4. **基础特征衍生**：生成是否归还 (`IS_RETURNED`)、借阅时长 (`LEND_DAYS`) 以及多粒度的时序特征（月、星期、小时）。

## 📈 探索性数据分析 (EDA)
通过多维度的数据可视化，挖掘校园读者的真实行为模式：
* **时序借阅规律**：发现显著的“寒暑假断崖效应”以及符合学生作息的午后与晚自习借阅双高峰。
* **用户与学科画像**：验证了财经类院校中“经济类（F类）”图书的绝对主导地位，同时发现研究生群体生均借阅活跃度极高。
* **深度交叉分析**：将中图分类号降维映射为 5 大类，结合时段分箱，生成了**学科-时段热力图**与**借阅趋势折线图**，为后续时间权重特征提供了现实依据。

## ⚙️ 特征工程 (Feature Engineering)
针对借阅数据的隐式反馈（Implicit Feedback）特性，构建了高维度的数学特征空间：
1. **图书内容向量化 (Item Profiling)**：联合图书标题 (`TITLE`) 与主题词 (`SUB`)，利用 TF-IDF 算法提取最多 5000 维的核心文本特征稀疏矩阵。
2. **创新性时间衰减因子 (Time Decay Factor)**：打破传统推荐算法中“历史权重等同”的弊端。采用半衰期指数衰减公式计算用户兴趣得分：
   $$Weight = e^{-\lambda \cdot \Delta t}$$
   其中 $\Delta t$ 为借阅行为距今的天数，有效捕捉读者随学期演进的兴趣漂移。
3. **交互矩阵构建**：聚合生成 User-Item 隐式兴趣得分矩阵，并应用 MinMaxScaler 进行归一化处理。

## 🧠 核心推荐模型简介 (Model Overview)
针对校园数据极度稀疏（万分之二的交互率）且存在严重“复本冗余”的长尾分布特点，本项目设计了一套融合内容语义与群体智慧的**混合推荐框架 (HybridModel)**：

1. **基于内容的推荐层 (Content-Based, CB)**：
   * **原理**：利用 TF-IDF 计算图书之间的余弦相似度，依据读者的历史借阅记录，推荐语义上高度相关的内容。
   * **作用**：提供极其稳定的基础推荐下限，有效缓解新书上架的“冷启动”问题。
2. **隐语义协同过滤层 (Latent Factor Model, LFM)**：
   * **原理**：摒弃传统 KNN 近邻算法（极易受同书不同码的复本干扰），采用 **截断奇异值分解 (Truncated SVD)** 对注入了“时间衰减因子”的隐式交互矩阵进行降维。将庞大的用户与图书矩阵映射至 50 维的稠密隐语义空间，通过隐向量内积预测潜在兴趣。
   * **作用**：大幅降低系统计算开销，抗稀疏性强，能够精准发掘读者深层的专业或兴趣圈层。
3. **混合决策与排序层 (Hybrid Strategy)**：
   * **原理**：通过 K-Fold 交叉验证与网格搜索，最终以最优超参数 $\lambda = 0.6$ 线性融合 CB 与 CF 的预测得分。
   * **机制**：在最终输出阶段引入了**动态历史屏蔽机制**（防止信息泄露）与**基于书名的哈希去重机制**（彻底解决推荐大量复本的业务痛点）。

### 模型离线评估结果 (Top-10 推荐)
基于留出法（盲测用户 20% 的未来真实借阅记录）的严格交叉验证，性能指标如下：

| 模型名称 | 准确率 (Precision) | 召回率 (Recall) | F1 值 (F1-measure) |
| :--- |:----------------|:-------------|:------------------|
| User-CF | 0.0068          | 0.0329       | 0.0113            |
| Item-CF | 0.0106          | 0.0458       | 0.0172            |
| Content-Based | 0.0118          | 0.0589       | 0.0197            |
| **HybridModel (本文)** | **0.0128**      | **0.0608**   | **0.0211**        |

*结论：混合推荐模型彻底排除了复本干扰，在准确率、真实召回率及 F1 综合指标上表现最为稳健，具有真实的知识发现价值。*

## 🌐 Web 可视化演示系统 (Demo)
> ![智能推荐系统界面](data/processed/images/visualization_recommendation_system.png)

为了直观验证推荐算法在实际业务场景中的应用效果，基于 **Flask** 框架开发了轻量级的交互式推荐系统 Web 界面。
* **核心特性**：
  * **用户偏好画像展示**：左侧动态渲染该读者历史借阅的 Top-10 图书及其时间衰减权重得分，增强系统可解释性。
  * **书名去重与复本过滤**：算法层加入了基于逻辑书名 (`TITLE`) 的哈希去重机制，避免因同书不同码 (`BOOK_ID`) 导致推荐列表出现冗余“复本”。
  * **动态悬浮窗 (Tooltip)**：结合全量数据挖掘，前端实现了鼠标悬停即可查看图书详细内容简介以及全校累计借阅总频次的功能，大幅提升交互体验。
